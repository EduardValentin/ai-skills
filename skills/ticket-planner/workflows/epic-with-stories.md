# Epic With Stories Workflow

Use when the user wants an epic and all of its stories planned together.

## Steps

1. Follow [single-epic.md](single-epic.md) through approval.
2. Propose a story list:
   - Story title
   - Brief scope
   - Vertical-slice outcome
   - Integration points covered
   - Suggested order
   - Dependency notes
3. Let the user add, remove, reorder, merge, or split stories.
4. For each approved story, follow [user-story.md](user-story.md).
5. Ask story-specific clarifying questions only when epic-level answers did not already resolve the decision.
6. Approve each story as a platform-neutral draft.
7. Summarize:
   - Epic draft
   - Story drafts
   - Integration coverage by story
   - Approved PRD updates, if any
8. If the user wants to publish the approved set, infer the issue tracker from project docs or ask, then switch to the relevant tracker-specific publishing skill.

## Quality Bar

Do not treat titles as stories. A story list is a planning artifact; each final story still needs a full approved description and acceptance criteria.
